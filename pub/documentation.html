Getting started

Get started with Spreadsheet.js by including the source code file as a script in the head of your HTML page:

<script defer type="text/javascript" src="Spreadsheet.js"></script>

Spreadsheets are generated by selecting an empty <div> by its id with the Spreadsheet function, where the spreadsheet will be generated. Then, the createSpreadsheet method must be called on the selected element, passing in the spreadsheet configuration objects. Similarly, adding any optional styles to the spreadsheet requires calling the addCellStyle or addCellStyleSheet methods on the selected element.


It should be noted that the <div> container must be selected by the Spreadsheet function before creating the spreadsheet. Similarly, the spreadsheet must be created before adding any styling rules.

Initializing an existing empty <div> element as a spreadsheet: 
const mySpreadsheet = Spreadsheet('#myContainer');
Creating a spreadsheet:

This spreadsheet has all the possible option configurations: a number to initialize the number of rows the spreadsheet will have, a boolean to indicate the spreadsheet will be saveable, a callback method receiving the spreadsheet's data and to be invoked when the "Submit" button is pressed, and a 2D JavaScript array to preload data into the spreadsheet. 

mySpreadsheet.createSpreadsheet(
  {
    bank: 'text',
    'cash ($)': 'number',
    'credit debt ($)': 'number',
    'investment portfolio ($)': 'number',
    city: 'text',
    'dated opened': 'date',
  },
  {
    rowCount: 4,
    persistent: true,
    submitCallback: (tableArray) => console.log(JSON.stringify(tableArray)),
    data: [
      ['TD', '10000', '1000000', '34535', 'Toronto', '2021-04-06'],
      ['RBC', '50000', '325345', '76868', 'Markham', '2020-06-22'],
      ['AmEx', '10000000', '5644', '76054', 'New York City', '2020-07-06']
    ]
  }
);
Adding a single styling rule:
mySpreadsheet.addCellStyle('10000', 'green');
Adding a styling rule object for multiple styles:
mySpreadsheet.addCellStyleSheet({ TD: '#64f547', RBC: '#5e87ff' });

Function(s)
Name Arguments Returned Description
Spreadsheet id: string (required) Spreadsheet object Creates the Spreadsheet object. This object has access to the createSpreadsheet, addCellStyle, and addCellStyleSheet methods to customize your spreadsheet.
Methods
Name Arguments Returned Description
createSpreadsheet Column object (required), Options object (optional) void Creates the actual spreadsheet accordingly to the names of the columns and its corresponding cell types (from the Column object), as well as any optional features (from the Options object). addCellStyle value: string (required), style: string (required) void Adds color styling to the spreadsheet, by coloring any cells containing the value argument text, and coloring it the style argument. The style argument can either be a string of a hexcode color or HTML-supported color such as 'red' or 'green'.
addCellStyleSheet Style object (required) void Adds styling to the spreadsheet exactly like addCellStyle, however it consumes a Style object, which is an object with keys as the cell value you want to style, and its corresponding value as the color you want to style it to. See the Style object API for more details.

Because each cell is an HTML input, it also supports cell data types such as 'date', 'month', 'time', etc. All HTML input types.

Options object

If rowCount, persistent, and data are all passed in as arguments and the user saved the spreadsheet data to localStorage already, the order of precedence goes: loading saved spreadsheet data -> using the preloaded 2D array table data -> just an empty spreadsheet with the row count.

That is, when there is saved spreadsheet data in the browser, it will load that saved user data first. If there is no saved data on localStorage, it will load data from the preloaded tableArray. If there is no data provided, then it will just load an empty spreadsheet using rowCount for the number of rows.

All key/value pairs are optional, as is the entire Options object.

Key Default Type Description
rowCount 1 number Represents the number of rows the spreadsheet will have.
persistent false boolean A boolean value on whether the spreadsheet will be saveable. If true, the spreadsheet will appropriately have the "Save" button for the user to save the spreadsheets data into the browser's localStorage.
data [] array A 2D JavaScript array that represents the data of a spreadsheet you wish to pre-load/initialize the spreadsheet with. Each sub-array represents a row, while each entry in the sub-array represents the corresponding cell's value.
submitCallback -- function A function that is invoked when the user presses the "Submit" button (button is present only when submitCallback was passed as an argument). The function is called and receives the spreadsheet's data in the form of a 2D array as an argument. This 2D array is formatted the exact same as the 2D array that can be passed in for the data parameter.
Style object
Key Value Description
[cell value]: string string This object's key/value pairs represent how cells with the key as its value text will be styled. The key represents the text that if cells contain it, it will be styled. And the value represents the hexcode or HTML color the cell will the colored with.
